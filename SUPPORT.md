# Support

Use the right channel so maintainers can respond efficiently.

## Bugs

Open a bug report with reproduction steps, affected protocol or UI area, environment details, logs, and Looking Glass output when relevant:

https://github.com/ParleSec/ProtocolSoup/issues/new/choose

## Questions And Ideas

Use GitHub Discussions for usage questions, design discussion, and early ideas that are not yet ready for an issue:

https://github.com/ParleSec/ProtocolSoup/discussions

## Contributor Help

For contribution workflow, setup, validation commands, and protocol extension guidance, start with:

- https://docs.protocolsoup.com/developers/overview/
- https://github.com/ParleSec/ProtocolSoup/blob/master/CONTRIBUTING.md

If you are planning a larger change, open a contribution proposal before starting implementation.

## Security Vulnerabilities

Do not open a public issue for security vulnerabilities. Follow the responsible disclosure process in [SECURITY.md](SECURITY.md).

## Production Use

ProtocolSoup is an educational protocol sandbox. It is not intended to be used as a production identity provider, authorization server, SCIM server, SSF transmitter, verifier, wallet, or SPIFFE authority.
